# 飞书关键词触发功能 - 使用说明

> 🤖 让机器人群聊中无需 @ 即可自动回复！

---

## 一、功能介绍

### 这是什么？

这是一个 OpenClaw 飞书插件的增强功能，让机器人在群聊中可以通过**关键词触发**自动回复，无需频繁 @ 机器人。

### 有什么用？

| 场景 | 使用前 | 使用后 |
|------|--------|--------|
| 群里叫机器人 | 必须 @机器人 | 说"傻妞"就行 |
| 日常聊天 | 机器人不干扰 | 只响应关键词 |
| 多人对话 | @ 来 @ 去很麻烦 | 自然对话体验 |

### 工作原理

```
用户发送消息
    ↓
检查是否包含关键词
    ↓
包含 → 触发机器人回复
不包含 → 保持安静
```

## 三、配置方法

### 基础配置

打开 `C:\Users\xiaohongPC\.openclaw\openclaw.json`，找到 `channels.feishu` 部分，添加 `triggerKeywords`：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_xxxxxxxxxxxxxxxx",
      "appSecret": "xxxxxxxxxxxxxxxx",
      "domain": "feishu",
      "groupPolicy": "open",
      "triggerKeywords": ["傻妞", "傻妞在吗", "叫傻妞"],
      "groups": {
        "oc_xxxxxxxxxxxxxxxx": {
          "requireMention": false
        }
      }
    }
  }
}
```

### 配置说明

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `triggerKeywords` | 触发关键词列表 | `["傻妞", "AI", "助手"]` |
| 配置位置 | 可放在全局或群组 | 全局/群组单独配置 |
| 优先级 | 群组配置 > 全局配置 | - |

### 配置示例

#### 示例 1：全局配置（所有群组共享）

```json
{
  "channels": {
    "feishu": {
      "triggerKeywords": ["傻妞", "傻妞在吗", "叫傻妞"]
    }
  }
}
```

#### 示例 2：群组单独配置

```json
{
  "channels": {
    "feishu": {
      "groups": {
        "oc_xxx": {
          "requireMention": false,
          "triggerKeywords": ["傻妞", "机器人", "AI"]
        }
      }
    }
  }
}
```

```powershell
  "C:\Users\xiaohongPC\.openclaw\workspace\backup\feishu-keyword-trigger\openclaw.json.original"
```

### 步骤 2：修改 bot.ts

**文件位置：** `C:\Users\xiaohongPC\AppData\Roaming\npm\node_modules\openclaw\extensions\feishu\src\bot.ts`

**查找代码（约 1023 行）：**
```typescript
if (requireMention && !ctx.mentionedBot) {
  log(`feishu[${account.accountId}]: message in group ${ctx.chatId} did not mention bot`);
```

**替换为：**
```typescript
// --- 关键词触发功能 ---
const triggerKeywords = feishuCfg?.triggerKeywords || [];
const hasTriggerKeyword = triggerKeywords.length > 0 && 
  triggerKeywords.some((kw: string) => ctx.content.includes(kw));

if (requireMention && !ctx.mentionedBot && !hasTriggerKeyword) {
  log(`feishu[${account.accountId}]: message in group ${ctx.chatId} did not mention bot and no trigger keyword`);
```

### 步骤 3：修改 channel.ts

**文件位置：** `C:\Users\xiaohongPC\AppData\Roaming\npm\node_modules\openclaw\extensions\feishu\src\channel.ts`

**查找代码（约 120 行）：**
```typescript
requireMention: { type: "boolean" },
groupSessionScope: {
```

**替换为：**
```typescript
requireMention: { type: "boolean" },
triggerKeywords: { type: "array", items: { type: "string" } },
```

### 步骤 4：修改配置文件

在 `C:\Users\xiaohongPC\.openclaw\openclaw.json` 中添加：

## 六、常见问题

### Q1: 修改后不生效？

**检查清单：**
- [ ] 网关是否已重启
- [ ] 配置格式是否正确（JSON 语法）
- [ ] 关键词是否匹配（区分大小写）
- [ ] 查看日志：`openclaw logs --follow`

### Q2: 机器人回复太慢？

可能原因：
- AI 模型响应时间（2-10 秒）
- 网络延迟
- 服务器性能

### Q3: 如何查看触发日志？

```powershell
openclaw logs --follow
```

搜索关键词：`trigger keyword`

### Q4: OpenClaw 更新后会怎样？

更新后修改会被覆盖，需要重新运行 `apply-patch.ps1`。

## 七、注意事项

1. **备份很重要** - 不要删除备份目录
2. **更新后重新应用** - OpenClaw 更新后需重新运行脚本
3. **性能影响小** - 每条消息增加约 0.1-0.5ms
4. **兼容性好** - 不影响其他功能和通道
